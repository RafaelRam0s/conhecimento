Display

Bloco:

A disposição de um bloco faz com que um elemento ocupe uma linha inteira só para ele

Desta forma o elemento que for um bloco cria uma linha só para ele, sem permitir elementos aos seus lados seja atrás ou após ele. Salvo excessão quando é utilizado float ou position para forçar que um elemento fique do lado

Mesmo se a largura do elemento for pequena, o display block continua a jogar o próximo elemento para baixo

Outra curiosidade é que se tiver um elemento com display block no meio do texto ele faz quebra de linha e ainda é afetado pelo text-indent do elemento pai. Assim como estou exemplificando no código com este parágrafo. Mas o mais curioso é que o Elemento que vem após esse span não recebe o text-indent.
Mesmo se eu forçar uma quebra de linha. Se bem que faz sentido já que para receber o text-indent deveria ser um novo parágrafo

Em linha

Disposição em linha permite que os elementos ajam como texto no posicionamento.

Um elemento em linha permite que num texto continue tudo sem alterar as linhas, como se fosse um complemento

Entre elementos irmãos ele se junta

E se adicionarmos um margin ele não funciona na vertical, e na horizontal ele funciona só na primeira linha se for um margin-left, e na última se for um margin-right

E se tivermos o padding ele irá acabar bloqueando a visualização de uma linha se ele for muito grande e tiver um background-color, pois ele torna o padding relativo a cada linha do texto e não do conjunto como o display block

Para ajustar a visualização, caso seje necessário ter esses efeitos, altere o tamanho entre as linhas (line-height) ou altere o tamanho da vertical (vertical-align) ou altere a fonte do texto (font-family). Mas nos 2 primeiros exemplos de como solucionar se tiver um elemento em linha antes ele acaba sofrendo o efeito, porém somente na última linha. Caso não visualize o exemplificado altere o tamanho da página

Lembrando que propriedades que alteram o tamanho do elemento podem não funcionar quando esta propriedade estiver ativa. E se a página for pequena um elemento irmão pode ser jogado para a linha de baixo e não ficar no mesmo bloco

Bloco em linha

Como o próprio nome indica é uma mistura das duas propriedades ele faz o elemento se comportar como um bloco, mas permite a presença elementos irmãos ao seu lado

Então isto é o que acontece quando possuimos elementos irmãos como display inline-block

Note que os elementos tendem a criar uma base, e nessa base é utilizada como posição para os elementos que forem ficar uns aos lados dos outros e forem menor. Então o elemento anterior ficou em baixo por ser menor

Mas temos como alterar isto com o vertical-align

Escondendo um elemento

O display: none; faz parecer que determinado elemeto nem existe, mas na real ele existe, mas na hora de rodar a tela o elemento que estiver com ele não irá ser contando para o ajustamento das outras e "desaparece", mas ele ainda continua ali e pode realizar/sofrer comandos

Já o visibility só modifica se um elemento deve aparecer visualmente ou não, é tipo um opacity com apenas dois valores (um de 0 e outro de 1)

Deixar um elemento escondido é feito para interações da tela, não deve ser usado simplesmente para posicionar um outro elemento

Itens Flex:

Os Flex Itens são os filhos diretos do Flex Container, lembrado que uma tag se torna um flex container a partir do momento que você definir display: flex. É possível que um Flex Item seja também um Flex Container, basta definir display: flex nele. Assim os filhos desse item também serão flex itens.

Define a habilidade de um flex item crescer. Por padrão o valor é zero, assim os flex itens ocupam um tamanho máximo relacionado o conteúdo interno deles ou ao width definido. Ao definir 1 para todos os Flex Itens, eles tentarão ter a mesma largura e vão ocupar 100% do container. Digo tentarão pois caso um elemento possua um conteúdo muito largo, ele irá respeitar o mesmo. Se você tiver uma linha com quatro itens, onde três são flex-grow: 1 e um flex-grow: 2, o flex-grow: 2 tentará ocupar 2 vezes mais espaço extra do que os outros elementos. OBS: justify-content não funciona em items com flex-grow definido.

Valor 0 só cobrirá o próprio conteúdo
