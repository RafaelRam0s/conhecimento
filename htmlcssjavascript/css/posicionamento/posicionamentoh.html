Posicionamento Estático:

O posicionamento estático está posicionado de acordo com o fluxo da página, onde seu elemento foi colocado é onde ele vai aparecer e ficar

Com os próximos valores do posicionamento é possível usarmos outras 4 propriedades, que são; top, right, bottom, left. Com essas classes podemos ajustar a sua posição para o lados da segunda dimensão

Posicionamento Fixo:

O posicionamento fixo deixa um elemento sobrepondo-se a todos os outros, e ficará prezo em um local da tela que for escolhido

Caso não configure onde na tela ele deve se enquadrar ele acaba por ficar fixo a partir de onde seu elemento apareceu, então lembre-se de configurar suas coordenadas com top, right,bottom e left. Caso o escreva muito em baixo ele poderá nunca ser visto se não for configurado corretamente.

Cuidado, suas outras propriedades ainda funcionam tal como o padding, margin e tantas outras então ao posicionar o elemento esteja ciente que essas propriedades irão atrapalhar no seu enquadramento

Ao deixar um elemento fixo ele não deixa um espaço vazio de onde ele deveria ter ficado, então basicamente nenhum elemento sofre seus efeitos de posicionamento, é como se ele houvesse sido inserido por último. Mesmo que haja outros elementos fixos um não interfere no espaço do outro então o que acaba acontecendo é que o último a ser escrito possui vantagem de sobreposição sobre o outro

Posicionamento Grudado:

O posicionamento grudado, gruda na tela quando a própria tela passa por seu elemento, mas se depois voltar por onde seria o local dele ele desgruda. Age como uma mistura de estático com fixed, estático enquanto não passar com a tela sobre ele, depois que passa é fixo, se voltar ele desgruda e fica estático novamente

Se definido um top indica que quando o topo da janela passar nele, ele irá grudar a partir da distância que foi definido em seu valor, se for bottom é quando for a parte de baixo, quando for right é a parte da direita e quando for left é a esquerda

Ele fica com sua propriedade parecida com o fixed enquanto ele tiver elementos irmãos para percorrer. Basicamente ele só funciona a parte do grudado nos elementos irmãos.

Para esta propriedade funcionar é preciso ao mínimo configurar um de seus lados, top, right, bottom ou left

Um posicionamento relative funciona parecido com o estatico, mas a crucial diferença é que este pode utilizar propriedades como top, right, bottom e left para ajustar sua posição

Uma coisa que é constante no posicionamento absoluto junto de um display: block; é que a partir da onde for definida a posição do elemento ele irá se esticar até atingir o fim lateral da página ou até onde for definido seu right.

Só o absoluto

1° Quando você só adiciona a posição absoluta o que ele faz é o elemento ficar destacado a partir de onde está. Isso indica que a partir de agora o que for escrito após esse elemento será escrito como se o elemento com posição absoluta não existisce, então na hora da página carregar o que acontece é como se o elemento de posição absoluta não existisce, então a página coloca o próximo elemento normalmente, mas como existe sim o elemento de posição absoluta ele adiciona depois.

Note que como ele adiciona depois, as margens dos outros elementos já foram definidas, então esse elemento absoluto sofre influência da margem do elemento irmão anterior a ele, mas só para se posicionar. Quer dizer que as margens verticais teriam efeito estre si neste caso então se a margem de baixo do elemento anterior ao elemento absoluto tiver 30px e a margem de cima do elemento absoluto tiver 30px também elas serão somadas totalizando em 60px de espaçamento

Somente os elementos anteriores ao elemento absoluto interferem nele se não tiver uma propriedade de posição como top, right, bottom e left, claro

Sobreposição/Profundidade

Quando temos um elemento sobre o outro, podemos ver que sua ordem de precedência escrita no código html é o que define se ele deve ficar por cima de outro elemento, quem for escrito por último possui mais prioridade.

Com o z-index podemos alterar essa ordem de precedência, então o elemento com maior valor fica com maior importância.

O z-index só funciona em elementos que não possuem o position como static

Por padrão elementos que possuem um mesmo valor de z-index resolve quem deve ser prioridade por quem foi escrito por último

Um elemento pai com z-index não deixa seus filhos se esconderem atrás dele com um z-index menor.

Mas se o elemento pai só tiver o position static ai é possível se você tornar o valor negativo

Se um elemento fixo tiver um z-index maior que um elemento pai os elementos filhos também vão ter um z-index menor que o do elemento fixo, mesmo se você configurar para ser maior isso só contará para os elementos irmãos
