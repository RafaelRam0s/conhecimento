O que é o function?

Imagine que você tem um código que pergunta ao usuário dois números e retorna depois a soma entre eles, aí você vai e cria um botão que faz isso através de alertas:

Ai pensa que você continua criando a sua página e quer chamar este mesmo código novamente, o que você acaba tendo que fazer então é recriar todas as intruções. Ai pensa que você coloca outras tantas vezes esse mesmo elemento.

Perceba que além da linha ficar gigante com um código tão simples, tivemos que repetir 1 código, várias vezes, com o intuito de ajudar a minimizar todo este problema nasce as functions

E não só para isso que foi feita as functions "organizar a visualização", foi feito para ser prático, imagine que agora você pare e pense "Putz ao invés de fazer a ação de somar vou querer que multiplique os números", nesta página colocamos até agora, 3 vezes o código, mas pense que fizessemos centenas, iriamos ter que ir em cada linha e mudar cada parte do código, você acaba tendo que ir caçar e modificar todos, onde no meio disso você pode acabar cometendo um erro e esquecendo de mudar uma parte

E pense em outra situação, você tem esse código de somar, agora pensa que você aproveita ele para criar outros dois códigos diferentes, mas que o utilizam em uma parte o código original.

Ai volto a supor se quisessemos alterar a soma por multiplicação, o código repetiu muito e para fazer uma alteração precisaria sair atrás de todas as linhas que usaram o código.

Agora vem a mágica da function, e se escrevessemos esse código apenas uma vez, aí para ativar ele usassemos apenas um nome (bem parecido com o que as variáveis fazem, mas ao invés de armazenar 1 valor, armazenaríamos vários códigos), facilitaria muito a manutenção, pois ele estaria apenas em um local

Para fazer uma função precisamos ter em mente a seguinte estrutura básica - function nomeDaFunction(parametros) { bloco }; - abordarei posteriormente sobre o que seriam os "parametros" que está escrito no código, por hora imagine que ele não existe, tudo começa avisando que você irá criar uma função, para isso você escreve "function" ao falar isso você está falando que uma função será gerada e que se for informado um nome, este nome será usado para ativar um "bloco", os blocos seriam as instruções que queremos que o código realize, todas as instruções devem ficam dentro das chaves da função, agora vamos mostrar um exemplo de como ficaria o nosso código inicial dentro de uma função:

function somarNumerosPorAlertas() { let num1 = Number(window.prompt('Informe um número:')); let num2 = Number(window.prompt('Informe outro número:')); window.alert('A soma entre os números informados é: ' + (num1 + num2)); }

Aí para ativar esse código escrito no bloco da função, usamos o nome da função - somarNumerosPorAlertas() - assim que esta linha é escrita aí que o código é rodado, é bem parecido com as variaveis, ao falarmos "function somarNum..." só estamos guardando uma ação, que poderá ser utilizada, dentro de um nome, mas ao falarmos depois de criado, o nome com o parênteses em seguida, temos o código funcionando:

O que são os parâmetros de uma function?

Pense que você tem uma função que pega 2 números informados pelo usuário e sorteia um número inteiro no intervalo dos números informados:

Sortear um número entre 2 valores


Veja que usamos o id dentro da função para pegar os valores e isso complica, porque se quisessemos fazer de novo o código do sorteio, teríamos que repetir o código e trocar o id de quem estamos pegando o valor, isso por si só perde a praticidade.

Para isso foi criado os parametros, pode se dizer que são um tipo de variável que funciona igual o - let -, mas é bem diferente como você passa um valor para ser armazenado.

Para entender como funcionam segue um exemplo com o código que temos para sorteio:

function sorteioAlcance(min, max) { var numero_max = max - min + 1 var numero_max_digitos = ('' + Math.abs(numero_max)).length var numero_sorteado = parseInt(Math.random() * (10 ** numero_max_digitos)) + 1 var numero_maximo_sorteado = '' for (var x = 0; numero_max_digitos - x != 0; x++) { numero_maximo_sorteado += 9 } numero_maximo_sorteado = Number(numero_maximo_sorteado) + 1 numero_sorteado = (numero_max * numero_sorteado) / numero_maximo_sorteado if (numero_sorteado % 1 > 0) { numero_sorteado = parseInt(numero_sorteado) + 1 } else if (numero_sorteado % 1 < 0) { numero_sorteado = parseInt(numero_sorteado) - 1 } numero_sorteado = numero_sorteado + min - 1 window.alert('O número sorteado foi: ' + numero_sorteado) }

Veja que na primeira linha exigimos 2 parametros, - function sorteioAlcance(min, max) { - que neste caso é o "min" e o "max", mesmo que uma função peça um parametro, você não possui obrigação de o usar, por exemplo: - function alertaSemParametro(parametro_qualquer) { window.alert('Olá Mundo!') } -

Tá criamos essa "pseudo-variavel" que chamamos de parametro, mas como passamos um valor para ele? A resposta está em quando chamamos a função - sorteioAlcance() - dentro do parenteses podemos passar um valor se existir um parametro, - sorteioAlcance(document.getElementById('teste1').value, document.getElementById('teste2').value) - e mesmo se na chamada você passar um terceiro parâmetro - sorteioAlcance(document.getElementById('teste1').value, document.getElementById('teste2').value, 'string teste') - , nada aconstecerá, pois a função simplesmente o ignora já que em nenhum momento ela exige que um terceiro parametro seja informado, parametros tambem aceitam valores de tipo primitivos, como número, string entre tantas outras coisas, em suma, parâmetro é um tipo de variável

Como usar o return na function?

Até agora fizemos com que a a function realizasce uma ação na página e com isso vimos o seu efeito, mas podemos também fazer com que uma função rode e retorne um valor, e com esse valor sendo retornado podemos trata-lo no meio de um código, exemplo:

function alertaPerguntaNome() { let nome = window.prompt('Qual o seu nome?') return nome } function alertaSejaBemVindo() { let nome_informado = alertaPerguntaNome() window.alert('Seja bem-vindo ' + nome_informado) }

Saiba que assim que o primiro return é escrito a função para de executar o que tiver nas linhas abaixo, então um return pode ser usado para quebrar a execução de um bloco, exemplo:

function returnQuebrandoCodigo() { return window.alert('Olá mundo!') }

Basicamente o return para a função de executar as próximas linha e caso seja informado um valor após ele, ele passa esse valor para quem o chamou.

Como fazer um parametro configurado com um valor inicial caso não informem?

É possível fazermos um parâmetro ter um valor inicial já pré-definido, esse valor serve para caso esqueçamos de informar ou caso seja mais convencional ter aquele valor fixo, onde em algumas raras exceções mechamos nele:

function sortearNumero(min = 1, max = 10) { var numero_max = max - min + 1 var numero_max_digitos = ('' + Math.abs(numero_max)).length var numero_sorteado = parseInt(Math.random() * (10 ** numero_max_digitos)) + 1 var numero_maximo_sorteado = '' for (var x = 0; numero_max_digitos - x != 0; x++) { numero_maximo_sorteado += 9 } numero_maximo_sorteado = Number(numero_maximo_sorteado) + 1 numero_sorteado = (numero_max * numero_sorteado) / numero_maximo_sorteado if (numero_sorteado % 1 > 0) { numero_sorteado = parseInt(numero_sorteado) + 1 } else if (numero_sorteado % 1 < 0) { numero_sorteado = parseInt(numero_sorteado) - 1 } numero_sorteado = numero_sorteado + min - 1 window.alert('O número sorteado foi: ' + numero_sorteado) }

E assim como falamos, apesar de ser um valor predefinido, podemos fazer ele ficar com outro valor, se passarmos o valor - undefined - ele vai considerar como se não tivessemos mechido no parametro, mas se colocarmos qualquer outro ele já passa a interpretar, exemplo: - sortearNumero(undefined, 2) - aqui falamos que o primeiro parametro vai ficar sem nada, e quando fica com nada ele assume, se definirmos para ele assumir, um valor inicial, já o segundo parâmetro moficamos ele e agora ele só vai até o número 2
