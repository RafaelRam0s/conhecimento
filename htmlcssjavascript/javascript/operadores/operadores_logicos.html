Operadores de atribuição.

Imagine que fazemos uma operação em que a variável que coleta um valor informado pelo usuario é a mesma variável que armazenará o resultado, exemplo:

function somarComDois() { let x; x = window.prompt('Informe um número inteiro:'); x = x + 2; window.alert('O seu número + 2 é igual a ' + x) }

Como na programação, escrever coisas repetidas vezes é algo considerado "não prático" existe uns símbolos que facilitam o nosso processo na hora de fazer operações, então para não repetirmos o x podemos fazer da seguinte forma - x += 2 - e com isso é a mesma coisa que escrever - x = x + 2 - e se escrevessemos - x += 2 + 3 - seria a mesma coisa que - x = x + 2 + 3 - além da adição vários outros operadores tem essa característica, são eles: >= <= += -= *= /= %= <<= >>= >>>= &= ^= |=

Operadores matemáticos complexos:

Como na programação existem muitos casos que só é preciso adicionar 1 na conta ou subtrair 1 foi criado o ++ e -- que basicamente fazem a ação de somar 1 e subtrair 1, respectivamente.

Se tivermos um código - let x = 5 - e depois falamos - x++ - o valor de x agora é 6, pois 5 + 1 = 6 e se fizermos depois um - x-- - ele ficaria com 5, pois 6 - 1 = 5.

@@@@@Por uma questão de lógica, escrever - ++x - e - x++ - ou - --x - e - x-- - é a mesma coisa, ambos irão adicionar e subtrair o valor 1 na variável, só que por uma questão de processamento eles diferem @@@@@ o x++ faz x + 1 enquanto ++x faz 1 + x @@@@ não sei o pq disso impactar (algumas pessoas falam que o sistema pode agir diferente de qual vc escolhe, não possuo exemplos para diferir ambos)

Operadores lógicos

Operadores lógicos são operadores que trabalham principalmente através dos booleanos. Por exemplo imagine que queiramos saber se a resposta do usuario é igual ao valor 5:

function respostaIgualCinco() { let x = window.prompt('Informe um número inteiro:') window.alert('Seu número é igual a 5? ' + x == 5) }

Os operadores lógicos são: != "diferente", !== "totalmente diferente", == "igual", === "totalmente igual", && "e" e || "ou".

Agora na prática vamos entender como cada um age:

O igual == pergunta se dois valores são iguais, por exemplo 5 == 5 é true, entenda que para o igual, '5' == 5 é true, pois ambos são 5, independentemente do seu tipo eles representam o 5, mas 5 == 55 é false, pq 5 e 55 são diferentes, assim como 'abóbora' == 'abóbora' é true, 'abóbora' == 'ameixa' é false

Mas se eu quiser validar tanto o valor quanto o seu tipo? Aí você pode usar o totalmente igual 5 === 5 é true, mas '5' === 5 é false.

Agora saiba que o diferente vai basicamente fazer o que o igual faz, mas totalmente o contrário o resultado, 5 != 5 é false, '5' != 5 é false, 5 != 6 é true, 'abóbora' != 'ameixa' é true

Totalmente diferente é o reverso do totalmente igual, '5' !== 5 é true, 5 !== 5 é false, 'teste' !== 'ola' é true.

O "e" lógico vai comparar duas operações booleanas uma a esquerda dele e outra a sua direita e a partir do resultado das duas gera um outro booleano, basta que um dos lados seja false para que ele retorne false:

true && true é true, false && true é false, true && false é false e false && false é false

O "ou" lógico vai comparar duas operações booleanas uma a esquerda dele e outra a sua direita e a partir do resultado das duas gera um outro booleano, basta que um dos lados seja true para que ele retorne true:

true || true é true, false || true é true, true || false é true e false || false é false
