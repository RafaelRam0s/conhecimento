Formulário

Formúlarios são campos em que o usuário pode interagir enviando uma resposta para a página

Um formulário é iniciado pela tagname - form - que serve para englobar todas as informações que o usuário for preencher

Na inserção é onde damos a opção para o usuário inserir uma informação, neste caso é aquele elemento branco digitável

No label possuimos o atributo - for - que indica a quem está conectado, isso serve para caso clique no label o campo input, seja ativado, o valor do - for - deverá ser o mesmo que o id do input para que funcione o redirecionamento

Quando enviamos uma informação pro javascript é diferente de quando enviamos uma informação para o servidor, no servidor é preciso ter algumas informações para que trate os dados, por conta disto quando for utilizar campos de informações é importante colocar um - name - no elemento que recebeu a informação

Inserção

@@@@@ Atributo - form - caso faça uma tagname form com alguns formulários e feche o conteiner, ainda é possível adicionar elementos no formulário utilizando o atributo form, sendo que a tagname form deva possuir um id e esse id deve ser informado como valor dentro do atributo form

@@@@@ Atributo minlength e maxlength - serve para inputs em que é possível escrever valores (como os tipos: text, number, email, password, search, tel e link). Configura em seu valor um número que corresponde respectivamente o minimo (minlength) e o maximo (minlength) de caracteres que deverá possuir a resposta do formulário

@@@@@ Atriburto min e max - aceita um valor mínimo e um valor máximo para a resposta de um input (funciona em number só permitindo aceitar valores compreendidos dentro desses dois atributos, funciona em range indicando entre quais valores o range atuará, funciona em date, datetime-local, month e week delimitando seguindo o padrão do value para configurar)

@@@@@ Parâmetro - multiple - permite inserir multiplas informações: no email permite acrescentar e-mails, você cadastra um e-mail ai ao final usa vírgula e espaço (, ) e então pode ser acrescentado outro e-mail e no file, permite selecionar e enviar mais de um arquivo

@@@@@ Atributo - step - permite pular intervalor de números especificados em seu valor, funciona em: number, range, date, datetime-local, month, time e week

@@@@@ Parâmetro - checked - permite uma opção já iniciar marcada, funciona nos tipos radio e checkbox

@@@@@ Atributo - placeholder - permite deixar um campo que ainda não foi preenchido pelo usuário com uma sombra de um texto, esse texto será referente ao que você colocou como valor, funciona em: text, search, url, tel, email, e password

@@@@@ Parâmetro - required - diz que é requerido que tal campo seja preenchido para que o envio do formulário possa ocorrer

@@@@@ Atributo - pattern - serve para configurar a forma com a qual você permite receber o formulário a partir do que for definido no valor,funciona em: text, date, search, url, tel, email, e password.
Em outras palavras o que ele faz é configurar valores que o formulário pode ou não pode ter

A expressão [a-z] reconhece todas as letras minúsculas.
A expressão [A-Z] reconhece todas as letras maiúsculas.
A expressão [A-z] reconhece todas as letras maiúsculas e minúsculas.
A expressão [A-Z0-9] reconhece todas as letras maiúsculas e números.
A expressão [a-e] reconhece as letras a, b, c, d e e.

A classe \w recupera todos os caracteres alpha numericos, ou seja, letras e números, mas não acentos ou caracteres especiais. É o equivalente a [a-zA-Z_0-9] A classe \W pega TODOS os caracteres que não seja alpha numericos, ou seja, pontuações e espaços. A classe \s é o equivalente [ \t\n\x0B\f\r] A classe \d é o equivalente [0-9] E as classses de negação. Lembrando que basta colocar o sinal de ^ logo depois do colchete inicial [: A expressão de classe [^y] reconhece qualquer caractere, exceto y. A expressão de classe [^a-e] reconhece qualquer caractere, exceto a, b, c, d e e. A expressão de classe [^\d] reconhece qualquer caractere, exceto 0, 1, 2, 3, 4, 5, 6, 7, 8 e 9. Brincando de validar um email Uma tarefa muito corriqueira é a validação de campos de e-mail. Sem entrar nas polêmicas (validar essas coisas sempre é chato), mas é legal para treinar o que você acabou de ler. A expressão para fazer a validação é essa:^\w*(\.\w*)?@\w*\.[a-z]+(\.[a-z]+)?$ Explicando: A expressão ^ indica o começo da string/linha. \w* pega qualquer caracteres alpha numericos, é o equivalente a [a-zA-Z0-9_]. O asterísco é quantitativo, detectando qualquer quantidade desses caracteres, iniciando no 0 e indo até o infinito. A expressão (\.\w*)? significa: parenteses inicia um agrupamento. A expressão \. detecta literamente um ponto .. A expressão \w* qualquer quantidade de caracteres alpha numéricos. O ponto de interrogação (?) é quantitativo: determina que o que vier imediatamente antes dele aparecer na expressão 0 ou 1 vez. Nessa expressão ele aparece duas vezes. O arroba seria o arroba do email mesmo… \w* que aparece depois do arroba já falamos várias vezes logo acima. \.[a-z] pega um ponto seguido de letras minúsculas. vai detectar algo como .com, .net, etc… + significa que o que estiver imediatamente antes dele precisa aparecer 1 ou mais vezes no termo. (\.[a-z]+): abrimos novamente um agrupamento com o parenteses. \. pega o ponto. A classe [a-z] seleciona qualquer letra minúscula. E o mais aparece novamente, dizendo que tudo aquilo que estiver antes dele deve aparecer pelo menos 1 vez E a expressão $ pra finalizar significa final da string.

@@@@@ Atributo - accept - especifica os tipos de arquivos que o servidor aceita (que podem ser enviados por meio de um upload de arquivo), só funcona com tipo file. Possíveis valores @@@@@:
image/ - já filtra pro usuário ver só imagens, mas ainda é possível do usuário configurar para mandar audio, o arquivo deve ser tratado no servidor

@@@@@ Atributo - autocomplete - o preenchimento automático permite que o navegador preveja o valor. Quando um usuário começa a digitar em um campo, o navegador deve exibir opções para preencher o campo, com base nos valores digitados anteriormente, funciona com os tipos: text, search, url, tel, email, password, datepickers, range, e color. Em alguns navegadores, pode ser necessário ativar uma função de preenchimento automático para que isso funcione (veja em "Preferências" no menu do navegador). Valores:
on - ligado
off - desligado

@@@@@ Parâmetro - disabled - desabilitado, torna um campo incapaz de ser alterado, cuidado que ainda podemos através do inspecionar tira-lo

@@@@@ Atributo - formaction - especifica a URL do arquivo que irá processar o controle de entrada quando o formulário for enviado. O atributo formaction substitui o atributo action do elemento - form -. O atributo formaction é usado com type = "submit" e type = "image".

@@@@@ Atributo - formenctype - especifica como os dados do formulário devem ser codificados ao enviá-los ao servidor (apenas para formulários com método = "post"). O atributo formenctype substitui o atributo enctype do elemento form. Nota: O atributo formenctype é usado com type = "submit" e type = "image".
Possíveis valores:
application/x-www-form-urlencoded
multipart/form-data
text/plain

@@@@@ Parâmetro - readonly - quando presente, especifica que um campo de entrada é somente leitura. Um campo de entrada somente leitura não pode ser modificado (no entanto, um usuário pode tabular até ele, destacá-lo e copiar o texto dele), adicione um value e veja que o texto aparece é selecionável, mas não alterável. Funciona com os tipos: text, number @@@@@

@@@@@ Atributo - size - configura o tamanho do input de caracteres visiveis, sendo o menor valor possível igual a 1. Funciona com os seguintes tipos: text, search, tel, url, email, e password.

O elemento input possui diversas formas dependendo do seu atributo type, segue alguns exemplos:

Tipo Texto:

O usuário pode digitar uma informação simples, que não requer muitas linhas para responder

Tipo Botão:

O botão é simplesmente um botão, não é passado nenhuma informação dele para o formulário assim como ele não altera nada no formulário

Utilizado para interações com o javascript

Tipo Caixa marcável:

Usado para poder marcar e desmarcar itens de uma lista





Tipo Cor:

Permite ao usuário selecionar um aspecto de cor

Tipo Data:

Configura uma data com dia, mês e ano, dependendo do navegador ele pode abrir em mês, dia e ano respectivamente

Mas os seus valores são passados na ordem de ano, mês e dia sendo separados pelo traço de subtração, exemplo: 2020-08-19 e em seu value para que tenha uma data selecionada desde o início é preciso seguir esse padrão

Tipo Data e hora:

@@@@@ Parece que foi removido ou ainda não possui suporte ele está sendo alterado por type="text"

Tipo Data e hora local:

Usado para pedir informações de data e hora

Para ajustar um valor inicial ele usa em ordem os seguintes parâmetros ano-mês-diaThora:minuto

Tipo E-mail:

Usado para quando pedimos um e-mail

Ele tem a vantagem de notar a presença de determinados elementos que compõem e não compõem um e-mail e então avisa o usuário caso tenha preenchido errado e tentado enviar o formulário, forçando o usuário a corrigir, pois caso não corrija ele não envia o formulário e redireciona o usuário para a caixa que está com erro e solta um title de aviso

@@@@@ Atributo - multiple - permite inserir mais de um e-mail, você cadastra um e-mail ai ao final usa vírgula e espaço (, ) e então pode ser acrescentado outro e-mail

Tipo Arquivo:

@@@@@ Você seleciona o elemento, então é retornado para você o seu diretório de arquivos, onde você poderá selecionar apenas um arquivo, ao clicar em abrir o elemento salva o nome do arquivo, @@@@@ mas não o caminho para o arquivo?

O label ativa o input de tipo file, então se quiser colocar um display none no input e editar o label ao seu gosto poderá ter um efeito bacana

@@@@@ Atributo - multiple - permite selecionar mais de um para envio

Tipo Escondido:

O tipo escondido serve para gerar ações que o usuário não precisa ver, um usuário comum não vê, mas se usar o inspecionar elemento ainda é possível ler seu elemento

Tipo Imagem:

Com o imagem podemos colocar uma imagem que ao ser clicada terá o papel de um submit, enviando assim os dados preenchidos

Além de retornar os dados preenchidos se a pessoa decide enviar as informações clicando na imagem, é acrescentado nas informações que serão transferidas a posição x e y de onde foi clicado na imagem para enviar o formulário

Tipo Mês:

É pedido para informar o mês por extenso e o ano

caso queira configurar um valor padrão tanto o mês quanto o ano devem ser informados por número, como: ano-mês

Tipo Número:

Aceita números e expressões matemáticas, como a letra "e" o ponto final e a vírgula, serve para pedir um valor numérico

@@@@@ Os atributos min e max configuram entre quais valores a aplicação irá aceitar para o envio do formulário, caso o valor não esteja compreendido entre os valores solicitados é retornado um title de aviso

@@@@@ O atributo - step - só deixará o formulário aceitar números que que são múltiplos de seu valor

Tipo Senha:

O tipo senha serve para esconder o que o usuário está digitando, mas só esconde visualmente, já que ele age como um tipo texto na hora de enviar a informação

Tipo Seleção:

Serve para dentro de uma gama de alternativas só ser possível de escolher uma opção

Para indicar que uma alternativa está relacionada a outra é preciso declarar um mesmo name para todas

Indique um value para que o formulário consiga interpetrar do que se trata
