Texto alternativo para Imagens:

O atributo - alt - se comporta de maneira um tanto diferente, pois se adicionarmos estilos a img, alguns são seguidos e outros não, por exemplo a largura e altura não são obedecidas, mas padding, margin, borda e por aí vai, são @@@@@. Como o alt não deveria aparecer, a página pode em alguns casos, ficar piscando durante o carregamento, pois ele acaba ocupando um espaço diferente do que o determinado. Recarregue a página neste quadro (onde está o problema de carregamento da imagem) e veja como este efeito de tela piscar pode ser ruim.

Tamanhos definidos desde o início:

Ao colocarmos uma imagem em nossa página é importante que seja definido os seus tamanhos, se a altura e a largura estiverem definidas, o espaço necessário para a imagem será reservado quando a página for carregada. No entanto, sem esses atributos, o navegador não sabe o tamanho da imagem e não pode reservar o espaço apropriado para ela. O efeito será que o layout da página será alterado durante o carregamento (enquanto as imagens são carregadas), o que pode acabar por fazer a sua página ficar piscando.

Para uma responsividade coloque por via do style o width como 100%, porém esteja ciente que essa ação pode causar em piscamento da tela enquanto a imagem carrega, veja o que é mais viável para a sua página responsividade ou conteúdo

Imagens no meio de textos

Caso queira colocar uma imagem no meio do texto, ajuste o vertical-align para que possa ficar bem posicionado só tome cuidado para não acabar colocando um tamanho muito grande e linha ficar com muito espaço

Imagens em base 64 (meia quatro):

Imagens em base 64 mais nada são do que imagens transformadas em códigos, visualmente falando nada muda, mas veja seu src no html. O base64 pode acabar por inutilizar qualquer outra configuração feita na própria tagname img. Meios alternativos para se configurar podem ser feito no css ao chamar um span que a contenha e referenciar a img como elemento filho (span img {width: 200px}

então coloque as configurações por css interno ou externo, mas não inline, se quiser também pode configurar no elemento pai de preferência não coloque nenhum outro atributo no elemento

Cuidados:

1° Se preocupe com direitos autorais

2° Ao definir os tamanhos para uma imagem veja se ela não ficará deformada ou ruim de visualizar

3° Não use um tamanho de imagem original maior do que você usará na página, pois o html pega o arquivo carrega e depois o reajusta para caber no seu elemento, por exemplo: temos uma imagem que baixei da internet de 2000x1900 (2000 pixels de largura e 1900 pixels de altura) e coloco na minha página somente para ficar em elementos de 500x475, então quando carregamos a página o html baixa todo o arquivo gigante e depois o reajusta para o tamanho do elemento, note que imagens maiores tendem a ser mais pesadas, logo consomem mais dados de internet, além de causar uma lentidão na hora de carregar as informações da sua página, pode ainda custar mais caro no seu plano de hospedagem, então vá num editor de imagens e diminua sua imagem

Imagens com legenda:

Também existem legendas de imagem que podem estar em outra parte do seu documento e serem referenciadas por um atributo, porém este atributo parece não ser utilizado por nenhum navegador, mas fica aqui a explicação dele: Seus valores agem como um href de link, é possível referenciar a legenda por url, id e caminho de arquivo, onde deverá conter explicações mais detalhadas sobre a imagem

Imagens diferentes de acordo com o tamanho de tela

Se você tiver imagens grandes, pode ser que em dispositivos menores como celulares o layout fique quebrado, para evitar isso podemos configurar diversas imagens para aparecem conforme o tamanho de tela

Para colocarmos imagens dependendo do tamanho da tela, é necessário que dentro do elemento picture tenha um ou mais elementos source, e um img
